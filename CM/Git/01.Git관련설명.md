# Git 사용 가이드

---

## Windows에서 Git 사용 시 한글 경로 문제 해결

### 문제 상황

Windows 사용자 계정 이름이 한글일 경우, 사용자 홈 디렉토리 경로에 한글이 포함됩니다. (예: `C:\Users\홍길동`). Git Bash 또는 다른 Git 클라이언트가 이 경로를 제대로 인식하지 못하여 저장소 클론, 자격 증명 저장 등에 문제가 발생할 수 있습니다.

### 해결 방법

Git이 사용할 `HOME` 환경 변수를 영문 경로로 명시적으로 재지정하여 이 문제를 해결할 수 있습니다.

1.  **영문 홈 디렉토리 생성**
    -   Git이 사용할 홈 디렉토리를 영문 경로에 새로 만듭니다.
    -   예: `C:\DEV\CM\Git\Home`

2.  **Git Bash 프로필 수정**
    -   Git 설치 경로의 `etc/profile` 파일을 텍스트 편집기로 엽니다.
        -   일반적인 경로: `C:\Program Files\Git\etc\profile`
    -   파일의 맨 아래에 다음 줄을 추가하여 `HOME` 환경 변수를 새로 만든 디렉토리로 설정합니다.

    ```bash
    # 사용자 홈 디렉토리 재지정 (한글 경로 문제 해결)
    export HOME="/C/DEV/CM/Git/Home"
    ```
    > **참고:** Git Bash는 Unix 스타일의 경로를 사용하므로, `C:\`는 `/C/`로 표기해야 합니다.

3.  **Git Bash 재시작**
    -   열려 있는 모든 Git Bash 창을 닫고 다시 실행하여 변경된 설정을 적용합니다.
    -   이제부터 Git은 새로운 홈 디렉토리를 기준으로 `~/.gitconfig`, `~/.ssh` 등의 파일을 읽고 쓰게 됩니다.
